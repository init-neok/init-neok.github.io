大模型自主搜索的黎明

2025年3月，两篇重量级论文同时发布。虽然名字惊人地相似，但它们的核心侧重点截然不同：
Search-R1 是"通用框架型"研究，R1-Searcher 是"能力塑造型"研究。

一图看懂：两篇论文的本质差别

对比维度 Search-R1 (UIUC/Google) R1-Searcher (RUC)
核心问题 如何把"搜索 + 推理"统一进 RL 框架 如何显式激励模型学会搜索
论文气质 框架化、方法论化 训练技巧化、能力塑形化
训练逻辑 单阶段 outcome-based RL 为主 两阶段 RL，先学会搜，再学会用搜到的信息答题
主要 RL 算法 PPO / GRPO 改造后的 Reinforce++，并分析 GRPO
奖励设计 很简单，主要看最终答案是否正确（EM） 分阶段奖励：先 retrieval reward，再 answer reward（F1）
任务覆盖 一般 QA + 多跳 QA，共 7 个数据集 更聚焦多跳 QA，共 4 个数据集
强调的创新 "search engine as environment" 的 RL formulation + retrieved token masking "search capability" 的显式激励 + 两阶段训练课程
最终风格 更像 Search-augmented RL framework 更像 RL-trained search agent recipe

动机对比：它们到底各自在解决什么问题？

Search-R1：建立通用范式

现有方法要么只是在输入后检索一次，要么让模型"看起来会搜"（Tool-use）但没在训练中真正学会。作者旨在解决：

  • 搜索行为怎样纳入 RL rollout？
  • 多轮检索与推理怎样交替进行？
  • 只靠 outcome reward 是否足够学出该能力？
  • 引入检索文本后，RL 如何保持稳定？

👉 问题意识更偏"框架与优化"

R1-Searcher：专攻能力塑造

现有 LRM 依赖内部知识导致幻觉，而已有方法泛化不足或成本高。作者旨在解决：

  • 怎么让模型主动意识到自己该搜？
  • 怎么让模型先掌握"搜索调用格式"和"习惯"？
  • 怎么不用 SFT cold start，仅靠 RL 就练出能力？

👉 问题意识更偏"能力激励与行为塑造"

方法流程：交替生成与 The Golden Trick

Search-R1 流程

问题 $x$
↓
<think> 推理
↓
若需要外部知识
↓
<search> query </search>
↓
<information> docs </information>
↓
继续 <think>
↓
<answer> final answer </answer>

R1-Searcher 流程

Stage 1: 学会"会搜、按格式搜"

鼓励模型发起检索。只看格式和动作，不看答案对错。

retrieval reward + format reward

Stage 2: 学会"搜得对、用得对"

继续推理 + 检索，但关注最终答案是否准确。

answer reward(F1) + format penalty

The Golden Trick: 检索内容 Loss 掩码

两篇论文同时发现的核心秘密（Retrieved Tokens Loss Masking）

在强化学习计算梯度时，必须将外部搜索引擎返回的文本排除在外。如果不 Mask，模型会尝试去"预测"维基百科的内容，导致极其不稳定的优化（Reward Hacking 或崩溃）。

Search-R1 实验特点：严谨控制

覆盖面广、控制严。不仅测了多跳，还有 General QA。统一设置 same retriever / same corpus / same training data / same pre-trained LLM 做公平比较。不仅和 RAG 比，还和 SFT、R1、rejection sampling 比。

主结果： 相比 RAG 基线，Qwen2.5-7B 平均相对提升 24%。它更像一篇"完整 benchmark + method validation paper"。

R1-Searcher 实验特点：能力验证

行为分析更强、工程味更重。不只是主结果，还分析了 RL vs SFT、不同 answer reward (F1 > EM)、数据难度分布、数据多样性等训练配方细节。还额外测试了在线搜索场景。

主结果： 在 Bamboogle 上切到 online search 后，比本地检索还能继续提升，且比 Search-o1 32B 更强。它更像一篇"训练 recipe + behavior study paper"。

最终结论：各自证明了什么？

1

Search-R1：建立稳健的强化学习范式

  • search engine 可以被正式并稳定地纳入 RL rollout。
  • 只用 outcome reward 也能学出有效的 search-and-reasoning。
  • retrieved token masking 是关键稳定因素。
  • PPO 和 GRPO 都可行，但 PPO 更稳定。

核心定调："search-augmented RL 是成立的，而且是可训练、可泛化、可分析的。"

2

R1-Searcher：打造自主搜索 Agent 的实战指南

  • 仅靠 RL（无 distillation/SFT cold start），也能把搜索能力学出来。
  • 两阶段奖励设计有效；F1 比 EM/CEM 更适合 answer reward。
  • 数据难度和数据多样性会显著影响 search behavior 的形成。
  • 学到的能力可以无缝迁移到 Online Search。

核心定调："如果你真想把模型训成会搜的 agent，奖励设计和训练课程很关键。"

💡 研究者视角的最终判断

如果你想写 Related Work 或 Method Comparison，Search-R1 更"规范"，它是框架型代表。

如果你想亲手复现或进行后续开发，R1-Searcher 更"有手感"，它提供了大量的实操踩坑经验（为何不搜、为何设阶段奖励等）。

不要直接比较谁的榜单分数更高，它们在用不同的方式，共同推开了大模型自主检索时代的大门。
